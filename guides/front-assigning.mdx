<Note>
  Parahelp is assigned Tickets like any teammate in Front.

  Once assigned, Parahelp works the conversation until it is resolved and only transfers when its configurations outline that it should or cannot resolve the issue.

  You can automatically assign Tickets to Parahelp using Front rules.
</Note>

## 1. Overview of your existing rules

<Steps>
  <Step title="Open your rules">
    In Front, open **Settings** (the gear icon at the bottom of the far-left icon rail), then go to **Automation → Rules and macros → Rules** tab.

  <Step title="Keep in mind Front has two layers of rules">
    Both can assign Tickets:

## 2. Set up a rule to assign new Tickets to Parahelp

You can set up Parahelp rules either by creating a completely new rule (option A) or adapting your existing rule(s) (option B).

### Option A: Create a new rule

<Steps>
  <Step title="Create rule">
    On the **Rules and Macros** tab, click **Create rule** (top right).

    <Frame>
      <img
        src="/images/image-51.png"
        alt="Image"
        title="Image"
        className="mx-auto"
        style={{ width:"97%" }}
      />
    </Frame>

    {/* SCREENSHOT */}
  </Step>
  <Step title="Create from scratch">
    In the **Rule templates** gallery, choose **Create from scratch**, then **Create linear rule**.

    <Frame>
      ![Image](/images/image-52.png)
    </Frame>

    {/* SCREENSHOT */}
  </Step>
  <Step title="Name the rule">
    Give the rule a **Name**, e.g. `Assign new Tickets to Parahelp`.
  </Step>
  <Step title="Set the When (trigger)">
    Click into the trigger and choose **Inbound message is received (new conversation)**. This fires when a new conversation comes in.
  </Step>
  <Step title="Set the If (conditions)">
    To assign **all** new Tickets, leave the inbox condition as **Any inbox** (or remove it). To scope it, see the optional section below.
  </Step>
  <Step title="Set the Then (action)">
    Click **\+ Add action**, choose **Assign to teammates**, then select **\[your Parahelp teammate\]**.

  </Step>
  <Step title="Create">
    Click **Create**.
  </Step>
</Steps>

### Option B: Adapt an existing rule

If you already have an assignment rule that's close to what you want, **recreate it as a new rule**:

- Open the original alongside so you can copy its settings across.
- Front has no one-click duplicate, so you'll rebuild it manually.
- In the **Rules** tab, click **Create rule**.
- Choose **Create linear rule**, re-enter the same **When**/**If** conditions as your existing rule.
- Set **Then → Assign to** teammate to **\[your Parahelp teammate\]**.

### Optional: Assign specific inboxes or channels to Parahelp

By default, with the inbox left as **Any inbox** and no channel condition, the rule assigns **all** inbound Tickets across all inboxes and channels. To narrow it:

- **By inbox:** in the **If** section, set **Conversation is in** and pick the specific inbox(es) Parahelp should handle.
- **By channel:** click **\+ Add condition**, add **Conversation comes from channels**, and select the channels Parahelp should handle (e.g. email, chat). At least one channel must be selected.

## 3. Before you set it live

Front has a few settings that can interact with or override your new rule.

- **Rule order:** rules run in the order listed. If an earlier rule has **Stop processing other rules** checked, it can halt before your Parahelp rule runs. Position the Parahelp rule accordingly.
  {/* SCREENSHOT */}
  <Frame>
    ![Image](/images/image-56.png)
  </Frame>
- **Assignment limits** (**Rules and macros → Assignment limits**): if your Parahelp teammate has a concurrent-assignment cap and it's reached, rule-based assignment can be skipped.
  - Set it to **No limit** if you want Parahelp to always receive assignments.
    <Frame>
      ![Image](/images/image-57.png)
    </Frame>
    {/* SCREENSHOT */}
- **Per-inbox "Assign to owner" rules** (**Settings → Inboxes → \[inbox\] → Rules → Assign to owner**): these auto-assign owners at the inbox level, independently of your team rule, and can override it. Check the inboxes Parahelp should handle.
  <Frame>
    ![Image](/images/image-58.png)
  </Frame>

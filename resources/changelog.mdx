<Update label="Tasks & Automations" description="July 8, 2026">
  The largest update to the Manager Agent to date. The Manager Agent can now delegate work across your team, and you can track everything that needs your attention in one place.

  <img
    src="/images/tasks-release.png"
    className="rounded-lg shadow-md"
  />

  **New**

  - **[Todos](/platform/tasks#todos).** When the Internal Agent identifies new work, like a knowledge gap to fill or a doc to update, it creates a todo with the relevant context and assigns it to the right user or team. Start any todo with one click.
  - **[Tasks page](/platform/tasks).** A board and list view of everything you're working on: todos waiting to be started, tasks in progress, in review, and done.
  - **[Inbox](/platform/tasks#inbox).** All notifications in one place: knowledge changes to review, new todos, and tasks that need your approval.
  - **[Automations](/platform/automations).** Continuous tasks are now Automations. The new Automations page gives an overview of your recurring and event-triggered tasks, with new templates.
  - **[Teams](/platform/settings#team-members).** Assign subject-matter experts to different work, like product knowledge gaps to one team and enterprise policy gaps to another. Manage teams in Settings → Teams.

  **Improved**

  - **Speed.** Tasks open faster, long conversations load more quickly, plus bug fixes and UI improvements across the platform.
</Update>

<Update label="Self-serve onboarding" description="June 19, 2026">
  Set up Parahelp yourself, without waiting on us.

  **New**

  - **Onboarding flow.** Connect your tools, upload your support policies, select the pages to sync from your help center, and import historical tickets, all in the platform.
  - **Onboarding task.** When setup is done, the Internal Agent researches everything you've connected, drafts your Customer Agent's configurations, and pings you in Slack with any questions.
  - **Ticketing system setup.** Connect your ticketing system yourself, then confirm which agent replies to customers.
</Update>

<Update label="Parahelp 3.0" description="June 18, 2026">
  You can now build your Customer Agent's tools by describing them in plain English. Every task is its own development environment, where the Internal Agent writes the tool, tests it against real tickets, and stages it for your approval.

  <iframe src="https://www.linkedin.com/embed/feed/update/urn:li:activity:7473425810381742080" title="Launching Parahelp 3.0" style={{ width: "100%", height: "600px", border: "0", borderRadius: "12px" }} allowFullScreen />

  **New**

  - **Build Customer Agent tools with natural language.** Prompt the Internal Agent to create and update your Customer Agent's tool files, test them, run commands, and search the web.
  - **Trigger the Internal Agent on events in your tools.** Continuous tasks (now [automations](/platform/automations)) can trigger on events in GitHub (issues created, PRs merged), Linear (issues created, status changed), and Notion (pages created or updated).
  - **Connect Parahelp to your Slack workspace.** Add the Internal Agent to any channel, have it search Slack messages, and post to specific people or channels.
  - **New connections.** Notion, GitHub, and Cursor join Linear and Slack, connected in one click.
  - **Template library.** Built-in templates for one-time and recurring Internal Agent tasks, inspired by how frontier teams use Parahelp.

  **Improved**

  - **Research.** The Internal Agent can search tickets by metadata (if enabled), create interactive graphs in reports, generate custom CSV files, and run one-off scripts.
  - **Visibility and control.** Version history for every configuration file, connection management, tool error notifications, and privacy controls, all on the platform.
  - **Grouped actions.** Actions are grouped in the interface, so you can review a task at a glance.
</Update>

<Update label="Long-running tasks" description="February 9, 2026">
  A new foundation for the Internal Agent, and the groundwork for the tool building that shipped in 3.0.

  **New**

  - **Long-running work.** The Internal Agent can run complex tasks of 30\+ minutes without intervention, like finding knowledge gaps, implementing fixes, and testing them in one go. Works from Slack too.
  - **Indefinite chats.** Chats no longer end. Publish multiple times within a single chat with persistent context.
  - **Activity.** Every action now appears in Activity, including follow-ups and Linear updates, not just configuration changes.

  **Improved**

  - **Faster configuration changes.** Approve memory and knowledge updates immediately, then test and optimize after release.
  - **Scheduled chats.** Natural language frequency descriptions, timezone selection, and custom notification tagging.
  - **Interface.** Redesigned chat components, better sidebar filtering, and intelligence improvements across the agent.
</Update>

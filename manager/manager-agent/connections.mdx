The Manager Agent works in your tools through connections. Connect a system once, and it can act there from any task. Manage them all on the "Connections" page in Settings.

## Built in

Always available, no connection needed:

- **Ticket research:** Search and analyze your historical support tickets, including by metadata (if enabled).
- **Customer Agent configurations:** Read, update, and test every configuration file.
- **Testing:** Run test simulations of your Customer Agent in a sandbox.
- **Web search:** Research anything on the web.
- **Files and scripts:** Generate CSV files, interactive graphs, and one-off scripts.

You can begin by simply prompting the manager agent in a new task.

## Custom connections

- **MCP:** Connect any MCP server to give your agents access to the tools it exposes, like a custom CRM, a data warehouse, or internal scripts.
- **API:** Connect a custom API endpoint, like your own backend or a third-party platform. The Manager Agent builds a tool that wraps the endpoint and tests it against real data before going live. For example, a "look up this customer in our internal admin panel" tool.

To set one up, prompt the Manager Agent: tell it what to connect and what you want the agents to do with it. It guides you through wiring it up, requests any secrets it needs, and creates the matching tool or script.
